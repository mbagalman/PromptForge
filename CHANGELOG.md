# Changelog

This file records meaningful changes to prompts in this repository. Bump rules and entry format are defined in [`CONTRIBUTING.md`](CONTRIBUTING.md#versioning-and-changelog).

The `version` field in each prompt's frontmatter is the source of truth for the *current* version; this file is the source of truth for *what changed between versions*.

## 2026-05-16

- **New `data-analysis/` directory** — adds a tabular-data analysis and modeling domain to the repo: three system prompts (one autonomous data analyst, one modeling engineer, one model auditor) plus a folder README. The directory pairs the analyst and modeling-engineer prompts (each produces a written report from an uploaded tabular file) with an auditor prompt designed to read the modeler's report skeptically — the modeling-engineer's output contract (TL;DR → Setup → Performance → What the model learned → Caveats → Files) maps directly onto the auditor's checklist. The new `data-analysis/README.md` documents the workflow, platform-setup specifics (notably the auditor requires code execution to be *disabled* while the other two require it enabled), and maintainer notes; it replaces a one-line `placeholder` file that landed with the initial prompt commits.
- **Three data-analysis prompts aligned to the canonical six-section template** — each gained YAML frontmatter and Role / Knowledge & sources / How requests are handled / Output contract / Guardrails and fallbacks structure matching `research/` and `legal/` prompts. Bodies existed pre-alignment inside a `## System Instructions` fenced-code-block wrapper alongside `## Design Summary`, `## Platform Setup Notes`, and `## Maintenance Notes` scaffolding; today's alignment promotes the canonical content out of the wrapper and relocates the platform-setup and maintenance content into the new folder README. Initial canonical version 1.0.0 for all three.
  - `data-analysis/data-analyst.md` — initial canonical version 1.0.0. Autonomous exploratory analysis of an uploaded tabular file. Five-phase internal workflow (Orient → Audit → Explore → Interrogate → Synthesize) producing a structured report (TL;DR → What's in the data → Findings → What I'd look at next → Caveats). Statistical posture rules (description-vs-inference, effect sizes alongside p-values, Simpson's-paradox vigilance, observational-data causal-language ban). Guardrails cover file/format issues, data-size and shape edge cases, question-and-data conflicts, and statistical edge cases. Tags: `data-analysis`, `exploratory-analysis`, `statistics`, `tabular-data`.
  - `data-analysis/predictive-modeling.md` — initial canonical version 1.0.0. Builds, evaluates, and delivers a tabular predictive model with XGBoost as the default plus explicit switch rules (small data → regularized linear, time-structured → temporal split, etc.). Eight-phase workflow (Frame → Inspect-and-split-BEFORE-engineering → Audit-for-leakage → Engineer-minimally → Build → Evaluate-honestly → Interpret → Hand-off) producing a fitted sklearn `Pipeline` artifact, a `MODEL_CARD.md`, and a report (TL;DR → Setup → Performance → What the model learned → Caveats and limitations → Files delivered). Two load-bearing guardrails: "split BEFORE engineering anything" prevents leakage at the workflow level; "stop and re-audit on suspiciously high metrics" catches it at the evaluation level. Tags: `data-analysis`, `modeling`, `xgboost`, `machine-learning`, `tabular-data`.
  - `data-analysis/predictive-model-auditor.md` — initial canonical version 1.0.0. Skeptical review of a written model report producing a short memo with one of three verdicts: "ready to use," "has fixable issues," or "should not be used as reported." Nine-section internal checklist (leakage, evaluation scheme, metric choice, subgroup performance, calibration, baseline, interpretation, model card, honest summary) used as thinking structure rather than output structure. Explicit "cannot run code" epistemic constraint — the auditor reviews what was reported, it does not empirically verify. Pairs with the modeling-engineer's output contract by design. Tags: `data-analysis`, `audit`, `model-review`, `machine-learning`.
- **Index integrations** — surface `data-analysis/` from the documented entry points so the directory is discoverable.
  - `PROMPTS.md` — added Data Analysis Prompts to Quick Navigation; added a Data Analysis Prompts table covering the three prompts; added a note pointing at the modeler↔auditor pairing and the platform-setup divergence around code execution.
  - `README.md` — added a Data analysis bullet to the Contents list, linking to `data-analysis/README.md`.

- **New `research/deep-research.md`** — initial canonical version 1.0.0. Adds the Deep-Research Brief Specialist: a brief-writing tool that turns an underspecified research question into a complete, well-scoped brief for an external deep-research agent (Claude Research, Gemini Deep Research, ChatGPT Deep Research). Operates in brief-writing mode only — it does not perform the research itself. Implements a fit-diagnosis step (declines requests that don't suit deep research and recommends the right alternative tool), a three-question scoping cap, a six-component brief template (Objective / Audience and use / Scope / Source priorities / Output / Analytical posture), and a menu of seven analytical-posture levers (confidence tiers, counterhypotheses, claim-level source attribution, quantitative posture, decision orientation, disagreement surfacing, quoted evidence). Drafting discipline rules ("specify the objective, not the method"; "always include both Prefer and Deprioritize source sections"; "mark open-ended dimensions explicitly"; "bound depth, do not maximize it") encode the 2026 consensus that deep-research planners degrade under procedural scaffolding. Sits alongside the existing `content-digest` → `fact-checker` → `red-team-analyst` verification pipeline rather than chaining with it — the chain it participates in is *brief writer → external deep-research agent*. Tags: `research`, `deep-research`, `briefs`, `scoping`, `prompt-engineering`. Adds `claude-md` to its `target_platforms` to declare it works as a repository-level instruction file too. `research/README.md` updated to add a "Brief-writing tool" subsection alongside the existing "Verification pipeline" three.
- `research/fact-checker.md` — 1.1.1 → 1.2.0 (minor): net-new audit capabilities and verification discipline aimed at deep-research compatibility. Major additions: (1) **Source priorities** subsection in Knowledge & sources with explicit Strongly Prefer / Acceptable / Deprioritize / Do-not-cite-as-authoritative tiers — addresses the default failure mode of research-capable models over-relying on highly-ranked but low-authority sources. (2) **arXiv and preprint verification** subsection — explicit guidance for distinguishing "could not verify the paper exists" from "verified that the paper does not exist," with the former marked Unverified rather than Contradicted. (3) **Verification discipline** subsection in How requests are handled: five rules — Unverified-is-not-Contradicted, don't-cite-an-unread-source, surface-disagreement, willingness-to-mark-a-high-volume-of-claims-Unverified, bound-depth (15–30 sources). (4) New "audit only empirical claims" rule in Claim triage priority — explicitly excludes subjective judgments, framing, and aesthetic choices from audit scope. (5) **Severity vs. confidence** constraint disambiguates the two scoring axes (severity = document impact; confidence = audit certainty). (6) Output contract additions: the 120–180-word Overall Reliability Verdict gets explicit content requirements (claim distribution, single-most-consequential-error); Citation Audit adds "Citation laundering instances"; Verification Log discipline tightens to "a source appears here only if it was read." (7) New **Ambiguous scope** guardrail — ask 1–2 clarifying questions before researching if the document's domain, intended use, or audience is unclear. (8) Default fallback expanded with the Unverified-vs-Contradicted clarification. Adds `deep-research-modes` to target_platforms and `deep-research` to tags. Subsection headings normalized from Title Case to lowercase to match other canonical prompts. Workflow content preserved; semantics tightened. Downstream consumers should re-validate if they key off any of the previously Title-Cased subsection names.

- **PROMPTS.md** — major refresh of the central index:
  - Expanded **Prompting Best Practices (2026)** section with four new/updated guides: `prompting-best-practices-2026.md`, `deep-research-best-practices-2026.md`, `good-prompting-for-beginners.md`, and `agents-md-best-practices-2026.md`.
  - **Research Prompts** table updated to reflect **Fact-Checker v1.2.0** (enhanced deep-research support, detailed verification workflow, hallucination/claim accumulation scans, platform notes).
  - Minor consistency and discoverability improvements across tables and navigation.
  - Last updated date refreshed to May 16, 2026.
